# 项目成果说明

## 项目名称

高校教师信息采集自动化工作流

## 项目定位

这是一个围绕高校官网教师信息采集、清洗、统计和导出的自动化项目。项目目标不是只做一个网页爬虫，而是尽量把“找网址、抓列表、补详情、清洗、汇总、日报复盘”串成一条可持续交付的工作流。

## 解决的问题

在真实业务中，早期流程高度依赖八爪鱼：

- 需要手动配置采集模板
- 需要先抓教师姓名和个人主页链接
- 需要再交给脚本补采详情页
- 需要人工处理学院误分类、邮箱缺失和重复数据

这个项目持续优化后的核心价值，是尽量跳过中间大量重复操作，把流程收敛为：

1. 找到教师列表页网址
2. 脚本自动识别教师个人主页
3. 自动打开个人主页补采信息
4. 自动过滤无效数据
5. 自动输出 CSV、日志和汇总 XLSX

## 当前成果

### 1. 采集流程更一体化

- 支持直接输入教师列表页 URL
- 支持从列表页中自动抽取教师姓名和个人主页
- 支持继续进入个人主页提取邮箱、职称、简介
- 支持单个教师主页 URL 直接补采

### 2. 数据质量更稳定

- 默认将“有个人邮箱”作为有效数据口径
- 过滤学院公邮、页脚公共邮箱
- 处理 `#` 代替 `@` 的反爬邮箱格式
- 处理跨 HTML 标签拆分邮箱
- 清洗姓名中的前缀/后缀职称
- 修正学校与学院误分类问题

### 3. 对复杂页面的适应性更强

- 支持普通 HTML 教师列表页
- 支持表格型导师页
- 支持卡片型教师页
- 支持部分个人列表页
- 支持部分 JS 动态渲染页
- 支持部分 Visual SiteBuilder 组件页
- 对图片形式的信息可通过 OCR 识别补采

### 4. 输出结果更贴近业务交付

- 自动生成学院 CSV
- 自动同步 `按院系汇总.xlsx`
- 自动生成日志，便于日报和复盘
- 支持统计某一天的有效数据数量
- 支持对已有结果再做清洗和补同步

## 典型实践收获

项目推进过程中，已经围绕真实业务问题补齐了多项规则：

- 解决部分高校页面 `403` 拦截
- 修复默认学院占位值带来的误填问题
- 修复多 URL 同行时后续链接漏识别问题
- 修复正文中毕业院校误判为当前学校的问题
- 修复公共邮箱被误当成个人邮箱的问题
- 修复列表页真实归属学院与手工输入学院不一致的问题
