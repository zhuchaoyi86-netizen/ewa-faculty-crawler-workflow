# 院校院系教师信息采集 SOP

## 1. 目标

每天稳定产出约 `800` 条教师信息，并整理成和模版一致的导入格式：

- `序号`
- `姓名`
- `职称`
- `邮箱`
- `学校`
- `学院`
- `采集人`
- `采集日期`
- `简介`
- `主页`

## 2. 推荐工作流

建议把采集拆成两段，不要一开始就追求“一套模板抓完所有网站”。

### 第一段：八爪鱼负责批量拿老师列表

适合抓：

- 学院官网中的 `师资队伍`
- `师资力量`
- `教师名录`
- `专任教师`

这一段的目标不是一次把所有字段抓全，而是尽快拿到：

- `姓名`
- `主页`
- `学校`
- `学院`

### 第二段：VSCode 脚本负责补详情和整理导入表

脚本负责：

- 访问老师主页
- 提取 `职称`
- 提取 `邮箱`
- 提取 `简介`
- 清洗字段格式
- 导出成模版要求的 CSV

这样做的好处是：

- 八爪鱼更适合处理分页、点击、批量列表
- 脚本更适合处理详情页清洗和格式统一
- 后续要自动化时，直接在脚本层继续扩展即可

## 3. 每日产能拆分建议

如果目标是每天 `800` 条，不建议按“学校”推进，建议按“院系任务”推进。

推荐节奏：

- 上午：准备 `20` 到 `30` 个院系任务
- 白天：批量抓列表页，拿到老师主页链接
- 下午/晚上：跑脚本补详情并导出
- 收尾：人工抽查 `5%` 到 `10%`

经验上，`800` 条更适合“半自动批量 + 人工校验”，纯手动容易累，纯模板化也容易被网站结构差异拖慢。

## 4. 任务池建议

建议在飞书表里维护一个“任务池”，一行对应一个院系：

- `学校`
- `学院`
- `学院官网`
- `教师列表页`
- `任务状态`
- `列表页结构类型`
- `备注`

任务状态建议统一成：

- `待找入口`
- `待抓列表`
- `待补详情`
- `待质检`
- `已完成`
- `跳过`

## 5. 八爪鱼 SOP

## 5.1 开始前先判断页面类型

先确认教师页面属于哪一类：

1. 卡片列表页
2. 表格列表页
3. 新闻列表式教师页
4. 点击姓名后进入详情页
5. 无详情页，信息直接在列表中

如果一个批次学校的网站结构相近，尽量放在同一天做，模板复用率会高很多。

## 5.2 每个院系的固定寻找路径

优先按下面顺序找：

1. `学校官网`
2. `学院官网`
3. `师资队伍 / 师资力量 / 教师队伍 / 教师名录 / 专任教师`

如果学院站没有独立教师页，再看：

- `机构设置`
- `系部设置`
- `导师介绍`
- `研究生导师`

## 5.3 八爪鱼列表页模板怎么设计

推荐最少抓取以下字段：

- `name`：教师姓名
- `profile_url`：教师主页链接
- `school`：学校名称
- `college`：学院名称
- `list_url`：当前教师列表页

如果列表页本身还能直接看到这些字段，也可以顺手抓：

- `title`
- `email`
- `intro_short`

但不要为了多抓这几个字段，把模板复杂度做得太高。核心还是先稳定拿到 `姓名 + 主页`。

## 5.4 八爪鱼的推荐步骤

1. 打开教师列表页
2. 选中单个教师卡片或单行教师信息
3. 创建循环列表
4. 提取姓名文本
5. 提取教师详情页链接
6. 处理分页
7. 导出 CSV

命名建议统一：

- 任务名：`学校-学院-教师列表`
- 导出文件名：`学校_学院_列表.csv`

## 5.5 分页处理建议

优先顺序：

1. 页码翻页
2. 下一页按钮
3. 无限下拉
4. 点击“查看更多”

如果分页很不稳定，不要反复调模板，先把当前页拿下，再人工补剩余页，避免卡住整天产能。

## 5.6 详情页要不要在八爪鱼里一起抓

建议初期不要。

原因：

- 不同老师主页结构差异大
- 很多详情页字段位置不固定
- 简介、邮箱、职称经常需要清洗
- 一个模板很难兼容所有学校

更稳的做法是：

- 八爪鱼只负责抓 `列表 + 链接`
- 脚本再批量处理详情页

## 6. 字段填写标准

为了后续导入稳定，字段口径最好统一。

### 姓名

- 只保留老师姓名
- 去掉前后空格
- 去掉 `教授`、`博士`、`硕导` 等附加称谓

### 职称

优先标准化为：

- `教授`
- `副教授`
- `讲师`
- `助理教授`
- `研究员`
- `副研究员`
- `实验师`
- `教师`

### 邮箱

- 优先保留学校官网公开邮箱
- 一个字段只保留一个主邮箱
- 没有则留空
- 不要猜邮箱

### 简介

- 优先取官网简介正文前 `80` 到 `200` 字
- 去掉过多空格、换行、导航文字
- 不要整页全量复制

### 主页

- 优先教师个人主页
- 如果没有单独主页，可退回教师列表页

### 采集日期

- 统一用当天日期

### 采集人

- 固定填你自己的名字，保持一致

## 7. 质检规则

每天建议至少做一次抽查。

重点检查：

1. `姓名` 和 `职称` 是否错位
2. `邮箱` 是否抓成别的联系信息
3. `简介` 是否混入导航栏、页脚、版权信息
4. `主页` 是否能打开
5. `学校`、`学院` 是否和当前任务一致

建议抽查比例：

- 新模板：`10%`
- 老模板复用：`5%`

## 8. 遇到异常时怎么处理

### 页面打不开

先排查：

- 学校官网是否临时故障
- 页面是否要求校内访问
- 是否存在验证码或频率限制

处理建议：

- 标记为 `待复查`
- 不要在一个站点上耗太久

### 列表页拿不到详情链接

处理建议：

- 先抓姓名和列表页地址
- 记录为“无详情页”
- 后续用搜索引擎或站内搜索补主页

### 详情页结构混乱

处理建议：

- 脚本尽量只抽取最稳定字段
- 特别乱的站点放入人工补录名单

## 9. 推荐目录结构

建议把项目整理成下面这样：

```text
易达威实习/
├── 模版/
├── 原始数据/
│   ├── 八爪鱼导出/
│   └── 待补详情/
├── 输出结果/
├── 日志/
├── scripts/
└── 采集SOP.md
```

## 10. 从今天开始的落地顺序

第一阶段先跑通，不追求全自动：

1. 选 `3` 个不同结构的院系网站做样板
2. 用八爪鱼抓出 `姓名 + 主页 + 学校 + 学院`
3. 用脚本补 `职称 + 邮箱 + 简介`
4. 导出模版格式
5. 人工抽查并总结结构规则

第二阶段再提升效率：

1. 把高频学校网站分结构归类
2. 八爪鱼模板复用
3. 在脚本里增加站点规则
4. 逐步提升自动化率
