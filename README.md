# 高校教师信息采集自动化项目

这是一个面向高校官网教师信息采集的自动化项目，目标是把原本依赖八爪鱼的半自动流程，逐步升级为以 Python 为主的可复用采集流程。

它不是单个“采集脚本”，而是一套可以连续交付的工作流：

- 输入教师列表页 URL 或已有 CSV
- 自动打开个人主页补采邮箱、职称、简介
- 自动过滤无效数据和学院公邮
- 自动输出学院 CSV、日志和汇总 XLSX

## 一眼看懂

- 适用场景：高校教师信息采集、导师库整理、学院官网师资数据导出
- 当前核心能力：列表页直采、个人页补采、清洗修正、有效统计、院系汇总
- 有效数据口径：默认要求必须有个人邮箱
- 工作流特点：尽量跳过八爪鱼，改为全 Python 一体化采集

## 快速开始

日常使用推荐直接运行交互入口：

```bash
python3 scripts/运行程序.py
```

如果你已经拿到一个学院教师列表网址，也可以直接运行主脚本：

```bash
PYTHONPATH=scripts python3 scripts/teacher_profile_pipeline.py \
  --list-page-url "https://example.edu.cn/faculty/list.htm" \
  --collector "你的名字" \
  --date "2026-06-15"
```

脚本会尽量自动识别真实学校与学院；如果页面里没有可靠邮箱，该老师不会进入最终有效导出。

当前项目已经支持：

- 输入教师列表页 URL，自动识别教师姓名与个人主页链接
- 自动打开教师个人主页，提取 `职称 / 邮箱 / 简介 / 学校 / 学院`
- 生成符合导入模板的 CSV
- 按日期同步汇总为 `按院系汇总.xlsx`
- 统计当天有效数据
- 对原始输出做清洗和格式修正
- 生成每日采集日志，便于复盘和汇报

项目适合：

- 高校教师信息批量采集
- 学院官网师资队伍页整理
- 导师信息标准化导出
- 需要持续每日交付的结构化采集任务

## 项目背景

这个项目的业务流程经历了几个阶段：

1. 早期主要依赖八爪鱼抓取教师列表
2. 中间阶段由脚本继续识别详情页并补全字段
3. 当前阶段已经可以对越来越多的教师列表页直接做全 Python 采集

也就是说，现在很多场景下已经不需要再先用八爪鱼逐个抓取 `姓名 + 个人链接`，只需要提供一个教师列表网址，脚本就可以自动完成后续采集、清洗、统计和汇总。

## 流程演进

这个项目并不是一次性设计完成的，而是在真实业务推进中不断修补和演进出来的。

### 阶段 1：八爪鱼抓列表，脚本补详情

最初的工作流是：

1. 用八爪鱼抓教师列表
2. 导出 `姓名 + 主页 + 学校 + 学院`
3. 再交给 Python 脚本访问教师个人主页
4. 补采 `邮箱 / 职称 / 简介`
5. 最终导出成标准 CSV

这个阶段解决了“批量抓列表”和“详情页结构不统一”的矛盾，但缺点也很明显：

- 八爪鱼步骤繁琐
- 模板维护成本高
- 不同学校分页和交互差异大
- 人工操作多，容易出错

### 阶段 2：规则补齐与数据清洗

随着业务推进，脚本不断补齐真实场景里的问题，包括：

- 跨 HTML 标签的邮箱提取
- `#` 代替 `@` 的邮箱反爬格式
- 使用浏览器风格请求头和 `Referer` 解决部分高校站点 `403`
- 过滤学院页脚公邮、公共邮箱
- 去除姓名中的前缀/后缀职称
- 处理括号人名、HTML 残留、UTF-8 文本残留
- 替换中英双版链接
- 通过 OCR 识别图片中的邮箱或个人信息

这个阶段的重点不是“多跑一点”，而是让数据质量稳定下来，减少无效记录、重复数据和学院误分类。

### 阶段 3：尽量跳过八爪鱼，直接 Python 采集

在不断补规则后，项目开始支持直接输入教师列表页网址，由脚本自动：

- 识别教师姓名
- 识别教师个人主页链接
- 逐个打开个人主页
- 抽取邮箱、职称、简介
- 识别真实学校和学院
- 分类输出到对应目录

目前已经补充了多类列表页解析：

- 普通教师列表页
- 表格型导师页
- 卡片型教师页
- 个人列表页
- 部分 JS 动态渲染页
- 部分 Visual SiteBuilder 组件页

这意味着现在很多高校站点可以不再走“八爪鱼先抓一轮”的老流程，而是直接用 Python 完成更一体化的采集。

## 当前能力

### 1. 教师列表页直采

支持多种教师列表页结构：

- 普通 HTML 教师列表
- 表格型导师列表
- 卡片型教师展示页
- 部分 Visual SiteBuilder 动态组件页
- 通过 JS 数组拼接教师卡片的页面

脚本会自动：

- 识别教师姓名
- 识别教师详情页链接
- 逐个打开个人主页
- 提取邮箱、职称、简介
- 根据页面和链接识别真实学校与学院

### 2. 有效数据规则

当前默认规则：

- “有效数据”必须包含邮箱
- 如果个人主页没有邮箱，则该老师不会进入最终有效导出
- 学院公邮、页脚公邮会尽量过滤，避免误当个人邮箱

### 3. 输出方式

默认输出包含：

- 单学院 CSV
- 当天 `按院系汇总.xlsx`
- 日志 JSON

典型字段：

- `序号`
- `姓名`
- `职称`
- `邮箱`
- `学校`
- `学院`
- `采集人`
- `采集日期`
- `简介`
- `主页`

## 核心脚本

### `scripts/teacher_profile_pipeline.py`

主采集脚本。

负责：

- 列表页识别
- 个人主页抓取
- 邮箱/职称/简介提取
- 学校学院识别
- 结果导出
- 汇总触发

### `scripts/运行程序.py`

交互式入口脚本。

适合日常使用，支持：

- 输入本地 CSV
- 输入教师列表页 URL
- 输入单个教师主页 URL
- 自动推断学校、学院和输出路径

### `scripts/清洗程序.py`

用于对已有采集结果做人工规则修正和清洗。

### `scripts/统计有效数据.py`

统计某一天已经导出的有效数据条数。

### `scripts/同步汇总xlsx.py`

用于把结果重新同步为汇总版 XLSX。

## 使用方式

### 方式 1：交互式运行

```bash
python3 scripts/运行程序.py
```

你可以直接输入：

- 本地 CSV 路径
- 教师列表页 URL
- 单个教师主页 URL

### 方式 2：直接运行主脚本

#### 输入教师列表页 URL

```bash
PYTHONPATH=scripts python3 scripts/teacher_profile_pipeline.py \
  --list-page-url "https://example.edu.cn/faculty/list.htm" \
  --school "学校名" \
  --college "学院名" \
  --collector "你的名字" \
  --date "2026-06-15"
```

说明：

- 即使 `school / college` 传错，脚本也会尽量根据链接和页面真实内容自动纠正

#### 输入本地 CSV

```bash
PYTHONPATH=scripts python3 scripts/teacher_profile_pipeline.py \
  --input "原始输入/2026-06-15/示例.csv" \
  --school "学校名" \
  --college "学院名" \
  --collector "你的名字" \
  --date "2026-06-15"
```

#### 输入单个教师主页 URL

```bash
PYTHONPATH=scripts python3 scripts/teacher_profile_pipeline.py \
  --profile-url "https://example.edu.cn/info/1234/5678.htm" \
  --collector "你的名字" \
  --date "2026-06-15"
```

项目保留了完整的每日日志思路，适合汇报和复盘。

### 实际遇到并解决的问题

1. 有些个人页中的信息是图片，不是文本
   - 通过 OCR 识别补采
2. 有些学校使用 `#` 替代 `@` 做邮箱反爬
   - 增加邮箱归一化规则
3. 有些页面跨 HTML 标签拆分邮箱
   - 增加跨标签邮箱识别逻辑
4. 有些站点直连返回 `403`
   - 使用浏览器风格请求头和 `Referer`
5. 有些公共邮箱会被误认为个人邮箱
   - 增加页脚公邮和机构邮箱过滤
6. 有些姓名带括号、前缀职称、后缀职称
   - 增加姓名标准化清洗
7. 有些页面既有中文页也有英文页
   - 对双版链接做替换与收敛
8. 有些数据会重复、无效或缺邮箱
   - 增加有效数据统计与过滤规则
