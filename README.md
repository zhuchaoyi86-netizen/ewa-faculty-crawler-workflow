# 高校教师信息采集自动化项目

这是一个面向高校官网教师信息采集的自动化项目，目标是把原本依赖八爪鱼的半自动流程，逐步升级为以 Python 为主的可复用采集流程。

当前项目已经支持：

- 输入教师列表页 URL，自动识别教师姓名与个人主页链接
- 自动打开教师个人主页，提取 `职称 / 邮箱 / 简介 / 学校 / 学院`
- 生成符合导入模板的 CSV
- 按日期同步汇总为 `按院系汇总.xlsx`
- 统计当天有效数据
- 对原始输出做清洗和格式修正
- 生成每日采集日志，便于复盘和汇报

项目适合：

- 高校教师信息批量采集
- 学院官网师资队伍页整理
- 导师信息标准化导出
- 需要持续每日交付的结构化采集任务

## 项目背景

这个项目的业务流程经历了几个阶段：

1. 早期主要依赖八爪鱼抓取教师列表
2. 中间阶段由脚本继续识别详情页并补全字段
3. 当前阶段已经可以对越来越多的教师列表页直接做全 Python 采集

也就是说，现在很多场景下已经不需要再先用八爪鱼逐个抓取 `姓名 + 个人链接`，只需要提供一个教师列表网址，脚本就可以自动完成后续采集、清洗、统计和汇总。

## 当前能力

### 1. 教师列表页直采

支持多种教师列表页结构：

- 普通 HTML 教师列表
- 表格型导师列表
- 卡片型教师展示页
- 部分 Visual SiteBuilder 动态组件页
- 通过 JS 数组拼接教师卡片的页面

脚本会自动：

- 识别教师姓名
- 识别教师详情页链接
- 逐个打开个人主页
- 提取邮箱、职称、简介
- 根据页面和链接识别真实学校与学院

### 2. 有效数据规则

当前默认规则：

- “有效数据”必须包含邮箱
- 如果个人主页没有邮箱，则该老师不会进入最终有效导出
- 学院公邮、页脚公邮会尽量过滤，避免误当个人邮箱

### 3. 输出方式

默认输出包含：

- 单学院 CSV
- 当天 `按院系汇总.xlsx`
- 日志 JSON

典型字段：

- `序号`
- `姓名`
- `职称`
- `邮箱`
- `学校`
- `学院`
- `采集人`
- `采集日期`
- `简介`
- `主页`

## 核心脚本

### `scripts/teacher_profile_pipeline.py`

主采集脚本。

负责：

- 列表页识别
- 个人主页抓取
- 邮箱/职称/简介提取
- 学校学院识别
- 结果导出
- 汇总触发

### `scripts/运行程序.py`

交互式入口脚本。

适合日常使用，支持：

- 输入本地 CSV
- 输入教师列表页 URL
- 输入单个教师主页 URL
- 自动推断学校、学院和输出路径

### `scripts/清洗程序.py`

用于对已有采集结果做人工规则修正和清洗。

### `scripts/统计有效数据.py`

统计某一天已经导出的有效数据条数。

### `scripts/同步汇总xlsx.py`

用于把结果重新同步为汇总版 XLSX。

## 推荐工作流

### 方案 A：教师列表网址直采

这是当前推荐流程。

1. 找到学院官网中的教师列表页或导师列表页
2. 把网址交给脚本
3. 脚本自动识别教师链接并打开个人页
4. 自动生成学院 CSV
5. 自动同步当天汇总 XLSX
6. 查看日志与有效数据统计

### 方案 B：CSV 输入补采

如果手头已经有 `姓名 + 主页` 的 CSV，也可以继续使用：

1. 准备 CSV
2. 跑主脚本补采详情
3. 导出结构化结果

## 使用方式

### 方式 1：交互式运行

```bash
python3 scripts/运行程序.py
```

你可以直接输入：

- 本地 CSV 路径
- 教师列表页 URL
- 单个教师主页 URL

### 方式 2：直接运行主脚本

#### 输入教师列表页 URL

```bash
PYTHONPATH=scripts python3 scripts/teacher_profile_pipeline.py \
  --list-page-url "https://example.edu.cn/faculty/list.htm" \
  --school "学校名" \
  --college "学院名" \
  --collector "你的名字" \
  --date "2026-06-15"
```

说明：

- 即使 `school / college` 传错，脚本也会尽量根据链接和页面真实内容自动纠正

#### 输入本地 CSV

```bash
PYTHONPATH=scripts python3 scripts/teacher_profile_pipeline.py \
  --input "原始输入/2026-06-15/示例.csv" \
  --school "学校名" \
  --college "学院名" \
  --collector "你的名字" \
  --date "2026-06-15"
```

#### 输入单个教师主页 URL

```bash
PYTHONPATH=scripts python3 scripts/teacher_profile_pipeline.py \
  --profile-url "https://example.edu.cn/info/1234/5678.htm" \
  --collector "你的名字" \
  --date "2026-06-15"
```

## 日志与日报

项目保留了完整的每日日志思路，适合汇报和复盘。

每次运行后都会生成对应日志文件，放在：

- `日志/`

日志中通常会记录：

- 处理条数
- 导出有效条数
- 被过滤的无效条数
- 哪些页面没有提取到邮箱
- 哪些链接抓取失败

这部分可以直接作为每天工作汇报的数据来源。

## 有效数据统计

统计某一天的有效数据：

```bash
python3 scripts/统计有效数据.py --date "2026-06-15"
```

适合做：

- 每日产量统计
- 工作汇报
- 项目阶段性复盘

## 清洗与修正

如果某批结果需要人工规则清洗，可以使用：

```bash
python3 scripts/清洗程序.py
```

适合处理：

- 姓名中的职称前后缀
- 页面误识别的学院邮箱
- 学院字段错误
- 批量格式修正

## 汇总同步

如果你修改了 CSV，希望重新生成汇总文件，可以使用：

```bash
python3 scripts/同步汇总xlsx.py
```

目前项目会尽量在主流程里自动同步，但这个脚本适合做补同步和二次整理。

## 已完成的典型改进

项目迭代过程中，已经完成了不少贴近真实业务的问题修复，例如：

- 默认学院不再固定成占位值

- 支持 Visual SiteBuilder 动态教师组件解析
